# Дмитрий Дрёмин / dimadr

Системный инженер, прошедший через несколько эпох развития вычислительной техники.

Начинал с Windows 3.x и NT-эпохи, занимался deployment engineering, автоматизацией и сопровождением систем задолго до появления терминов DevOps и Platform Engineering в их современном виде.

В сообществе OSZone был известен как `dimadr`, работал с Windows servicing, UpdatePack-интеграцией и unattended/deployment workflows.

## Текущий фокус

* AI automation & orchestration
* Инфраструктура и интеграция workflow
* Linux / Debian systems
* SSH, networking, VPN
* Надёжность и operational simplicity
* Практическая автоматизация для людей

## Инженерный подход

Предпочитаю:

* практичные системы вместо хайпа
* простоту вместо overengineering
* понимание вместо слепой абстракции
* инструменты, которые уважают инженеров и пользователей

## Опыт и контекст

За годы прошёл через несколько технологических переходов:

* desktop computing
* ранний интернет
* Windows deployment ecosystem
* Linux infrastructure
* cloud и automation workflows
* современные AI tooling и orchestration

Инструменты меняются.
Цель — нет:

создавать системы, которые уменьшают хаос и помогают людям работать эффективнее.
